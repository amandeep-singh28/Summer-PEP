****LangChain: Simplifying LLM Provider Switching****
If we change the LLM provider than we need to change our code also(OpenAI, Anthropic)
For example:-
->Using OpenAI
from openai import OpenAI

client = OpenAI()

response = client.chat.completions.create(
    model="gpt-5",
    messages=[{"role": "user", "content": "Hello"}]
)
print(response.choices[0].message.content)

-> Using Anthropic
from anthropic import Anthropic

client = Anthropic()

response = client.messages.create(
    model="claude-sonnet-4",
    max_tokens=100,
    messages=[
        {"role": "user", "content": "Hello"}
    ]
)
print(response.content[0].text)

->Solution: LangChain Models
We have models parameter in our LangChain -> It makes our life easy by providing a common interface

(i)OpenAI with LangChain
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(
    model="gpt-5"
)

response = llm.invoke("Hello")

print(response.content)

(ii)Anthropic with LangChain
from langchain_anthropic import ChatAnthropic

llm = ChatAnthropic(
    model="claude-sonnet-4"
)

response = llm.invoke("Hello")

print(response.content)

Two types of token:
    Input and output token

Token(I/p) < Token(O/p)

(iii)temperature
The temperature parameter controls the randomness or creativity of the model's output.
    Low temperature → More deterministic and focused answers
    High temperature → More creative and diverse answers

****Embedding Models****
Models are of two types:-
    Text(OS & Closed) -> Input as text and output as Text
    Embedding -> Input as text but output as vectors

Work:-
(i)Create vectors for all of the paragraphs(Documents)
(ii)Create vector for query
(iii)Find the closest document vector for the query vector
(iv)Closest would be the answer

dimensions = 500, length of each vector (Even if there is a single word, the single word will represent 500 length)

****Chain****
What are we doing?
Pipeline: Input → Prompt → Model → Parser → Output
We first provide an input to the application.
That input is converted into a prompt (either a Single Message Prompt or a Multi-Message Prompt).
The prompt is then sent to the LLM (Model).
The model generates a response.
The response is passed to an Output Parser, which formats or extracts the required information.
Finally, the parsed result is displayed as the output.

Here are the steps in a short and easy-to-remember format:

(i)Import the required libraries.
(ii)Create the Prompt Template (ChatPromptTemplate).
(iii)Create the Model (ChatGoogleGenerativeAI).
(iv)Create the Output Parser (StrOutputParser).
(v)Invoke the Prompt using chat_prompt.invoke() to generate the final prompt.
(vi)Invoke the Model using model.invoke() to get the model's response (AIMessage).
(vii)Invoke the Parser using parser.invoke() to extract the text from the model's response.
(viii)Print the final output.
